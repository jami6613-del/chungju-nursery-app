# 가입승인·닉네임 관련 확인·수정 방법

Supabase 대시보드에서 아래 순서대로 하면 됩니다.

---

## ★ 닉네임 설정에서 "Could not find the function public.update_my_name" 오류가 날 때

**닉네임 입력 후 확인**을 눌렀는데 위와 같은 빨간 에러가 나오면, **닉네임 저장용 함수**가 아직 Supabase에 없어서입니다. **한 번만** 아래 SQL을 실행하세요.

1. Supabase **SQL Editor** → **+ New query**
2. 아래 **전체**를 복사해서 붙여넣고 **Run** 클릭

```sql
CREATE OR REPLACE FUNCTION public.update_my_name(new_name TEXT)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  UPDATE public.users
  SET name = NULLIF(TRIM(new_name), '')
  WHERE id = auth.uid();
END;
$$;

GRANT EXECUTE ON FUNCTION public.update_my_name(TEXT) TO authenticated;
```

3. 실행이 끝나면 **앱**에서 다시 **닉네임 설정** 화면으로 돌아가서 **테스터** 입력 후 **확인**을 눌러 보세요.

---

## ★ 승인했는데 해당 계정으로 로그인이 안 될 때

**가입승인은 했는데**, 그 계정(jami6613@naver.com 등)으로 로그인하면 "로그인에 실패했습니다"가 나오는 경우, **Supabase 쪽 "이메일 확인"** 때문일 수 있습니다.

### 1) 이메일 확인 끄기 (앞으로 가입하는 사람은 바로 로그인 가능)

1. Supabase 대시보드 → **Authentication** → **Providers** → **Email** 이동
2. **「Confirm email」** 항목을 찾아서 **끄기(비활성화)**
3. **Save** 클릭

이렇게 하면, **이후에 가입·승인한 사람**은 이메일 링크 없이도 로그인할 수 있습니다.

### 2) 이미 가입한 계정(jami6613@naver.com)이 로그인되게 하기

**방법 A – 메일에서 확인 링크 누르기**  
jami6613@naver.com 메일함(스팸 포함)에서 **Supabase**에서 온 **「Confirm your mail」** 같은 제목의 메일을 찾아, 안에 있는 **확인 링크**를 한 번 눌러 주세요. 그다음 다시 로그인해 보세요.

**방법 B – Supabase에서 이메일 확인 처리하기**  
Supabase **SQL Editor**에서 **새 쿼리**를 연 뒤, 아래를 **그대로 붙여넣고 Run** 하세요.  
(실행이 거부되면 해당 프로젝트에서는 방법 A만 가능한 경우입니다.)

```sql
UPDATE auth.users
SET email_confirmed_at = now()
WHERE email = 'jami6613@naver.com' AND email_confirmed_at IS NULL;
```

실행 후 **다시 앱에서 로그인**해 보세요.

---

## ★ 승인 버튼을 눌러도 계속 "승인 대기"로 남을 때

**승인**을 눌렀는데 목록에서 사라졌다가 다시 나타나거나, 해당 계정으로 로그인하면 여전히 "승인 대기 중"이라고 나오는 경우, **승인 처리용 RPC**를 한 번 설정해 두세요.

1. Supabase **SQL Editor** → **+ New query**
2. 아래 **전체**를 복사해서 붙여넣고 **Run** 클릭

```sql
CREATE OR REPLACE FUNCTION public.approve_user_by_admin(
  target_user_id uuid,
  new_role_level smallint
)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  IF (SELECT u.role_level FROM public.users u WHERE u.id = auth.uid()) IS DISTINCT FROM 0 THEN
    RETURN;
  END IF;
  UPDATE public.users
  SET is_approved = true, role_level = new_role_level
  WHERE id = target_user_id;
END;
$$;

GRANT EXECUTE ON FUNCTION public.approve_user_by_admin(uuid, smallint) TO authenticated;
```

3. 실행이 끝나면 **앱**에서 다시 **가입승인** 메뉴로 들어가서 해당 계정을 **승인**해 보세요. 이제 승인 내용이 DB에 제대로 저장되고, 그 계정으로 로그인할 수 있어야 합니다.

**이미 "승인 대기"로 막혀 있는 계정을 지금 당장 풀고 싶다면** (예: jami6613@naver.com), 위 RPC 실행 후 앱에서 한 번 더 승인해도 되고, 또는 Supabase SQL Editor에서 아래를 실행해도 됩니다. (한 번만 실행)

```sql
UPDATE public.users SET is_approved = true, role_level = 3 WHERE email = 'jami6613@naver.com';
```

실행 후 해당 계정으로 로그인해 보세요. (등급을 Lv1/Lv2로 바꾸려면 `role_level = 1` 또는 `2`로 넣으면 됩니다.)

---

## ★ 데이터는 있는데(is_approved = false) 앱에서만 안 보일 때

**확인 쿼리에서 1줄 나오고 is_approved가 false인데**, 앱 가입승인 메뉴에는 여전히 "승인 대기중인 가입 요청이 없습니다"라면, **아래 SQL을 한 번만 실행**하세요.

1. SQL Editor에서 **+ New query** 클릭
2. **아래 네모 안 글 전체**를 복사해서 붙여넣고 **Run** 클릭

```
CREATE OR REPLACE FUNCTION public.get_pending_approval_users()
RETURNS TABLE (
  id uuid,
  email text,
  name text,
  role_level smallint,
  is_approved boolean,
  created_at timestamptz
)
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  IF (SELECT u.role_level FROM public.users u WHERE u.id = auth.uid()) IS DISTINCT FROM 0 THEN
    RETURN;
  END IF;
  RETURN QUERY
  SELECT u.id, u.email, u.name, u.role_level, u.is_approved, u.created_at
  FROM public.users u
  WHERE u.is_approved = false
  ORDER BY u.created_at ASC;
END;
$$;

GRANT EXECUTE ON FUNCTION public.get_pending_approval_users() TO authenticated;
```

3. 실행이 끝나면 **앱**에서 **관리자 → 가입승인** 메뉴를 **새로고침**(다시 들어가기) 해 보세요. 목록이 보여야 합니다.

---

## 1단계: Supabase SQL Editor 열기

1. 브라우저에서 **Supabase** 사이트 접속 후 로그인
2. **내 프로젝트** 선택
3. 왼쪽 메뉴에서 **「SQL Editor」** 클릭
4. **「+ New query」** 버튼 클릭 (새 쿼리 창이 열림)

→ 여기서부터는 **이 창에 글을 붙여넣고 Run 버튼만 누르면 됩니다.**

---

## 2단계: 먼저 “확인” 쿼리 실행

**아래 글 전체를 복사**해서 SQL Editor **빈 칸에 붙여넣기** 한 다음, **Run** 버튼 누르기.

```
SELECT id, email, created_at FROM auth.users WHERE email = 'jami6613@naver.com';
```

- **결과가 1줄 나오면** → 가입은 된 상태. **3단계로** 가면 됨.
- **결과가 0줄(비어 있으면)** → 이 이메일로는 가입이 안 된 것. 앱에서 같은 이메일로 다시 가입한 뒤, 다시 2단계 실행.

---

## 3단계: “동기화” 쿼리 실행 (승인 대기 목록에 넣기)

**2단계에서 1줄 나왔다면**, 이번에는 **아래 글 전체를 복사**해서  
SQL Editor에 **새 쿼리(New query)로 붙여넣고** **Run** 한 번 누르기.

(기존 2단계 쿼리 지우고, 아래만 넣어도 됨.)

```
-- 동기화 함수 (한 번만 만들면 됨)
CREATE OR REPLACE FUNCTION public.sync_auth_users_to_public()
RETURNS integer
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public, auth
AS $$
DECLARE
  inserted_count integer := 0;
  r record;
BEGIN
  FOR r IN
    SELECT au.id, au.email
    FROM auth.users au
    WHERE NOT EXISTS (SELECT 1 FROM public.users pu WHERE pu.id = au.id)
  LOOP
    INSERT INTO public.users (id, email, role_level, is_approved)
    VALUES (r.id, r.email, 3, false)
    ON CONFLICT (id) DO NOTHING;
    inserted_count := inserted_count + 1;
  END LOOP;
  RETURN inserted_count;
END;
$$;

-- 동기화 실행
SELECT public.sync_auth_users_to_public() AS "추가된_승인대기_수";
```

- **결과에 「추가된_승인대기_수」가 1 이상**이면 → `jami6613@naver.com`이 승인 대기 목록에 들어간 것. **4단계로** 가면 됨.
- **결과가 0이면** → 이미 `public.users`에 있다는 뜻. **3-1단계**로 가서 “승인 대기”로 바꿔 주기.

---

## 3-1단계: “추가된 수 0”일 때 (이미 있는 계정을 승인 대기로 만들기)

**3단계에서 추가된_승인대기_수가 0이었다면** 아래를 실행하세요.

**① 먼저 확인**  
아래를 복사해서 SQL Editor에 붙여넣고 **Run** 하세요.

```
SELECT id, email, is_approved FROM public.users WHERE email = 'jami6613@naver.com';
```

- **결과가 0줄** → 계정이 `public.users`에 없는 상태. 이 경우는 별도로 알려 주세요.
- **결과가 1줄**이고 **is_approved가 true** → 이미 “승인됨”으로 되어 있어서 목록에 안 보이는 것입니다.

**② 승인 대기로 바꾸기**  
`is_approved`가 true로 나왔다면, 아래를 복사해서 **Run** 하세요.  
(다시 “승인 대기”로 만들어서 가입승인 목록에 보이게 하는 것입니다.)

```
UPDATE public.users SET is_approved = false WHERE email = 'jami6613@naver.com';
```

실행 후 **앱 → 관리자 → 가입승인** 메뉴를 새로고침해 보세요.

---

## 4단계: 앱에서 확인

1. **관리자 계정**으로 앱 로그인
2. **관리자 메뉴 → 가입승인** 들어가기
3. `jami6613@naver.com`이 **승인 대기** 목록에 보이면 성공.

---

## 정리

- **「한 번에 실행」** = 그 단계에 적어 둔 **글 전체를 한 번에 복사**해서 SQL Editor **한 칸에 붙여넣고**, **Run 한 번** 누르면 된다는 뜻입니다.
- 2단계와 3단계는 **서로 다른 쿼리**라서, **2단계 실행할 때는 2단계 글만**, **3단계 실행할 때는 3단계 글만** 넣고 Run 하면 됩니다.
