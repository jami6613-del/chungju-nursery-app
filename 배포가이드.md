# 충주 친환경 육묘장 앱 배포 가이드 (쉬운 버전)

GitHub + Vercel로 배포한다고 가정하고, **처음부터 차례대로**만 따라 하면 됩니다.

---

## 이게 뭐냐면

- 우리 앱은 **Supabase**라는 서버(로그인, DB)를 이미 쓰고 있어요.
- **배포**란 = 우리가 만든 화면(React 앱)을 **Vercel**이라는 무료 사이트에 올려서,  
  **주소 하나(예: https://우리앱.vercel.app)** 로 누구나 접속하게 만드는 거예요.
- 그래서 **폰에서 그 주소만 열면** 같은 앱을 쓸 수 있어요.

---

## 0단계: “환경 변수”가 뭔지

우리 앱이 **Supabase랑 대화하려면** 두 가지를 알아야 해요.

1. **Supabase 주소** (URL) — “어디로 요청 보낼지”
2. **anon key** — “이 앱이 접속해도 된다”는 허용 키

이 두 개를 **이름 + 값**으로 적어두는 걸 **환경 변수**라고 부릅니다.

- **로컬(지금 PC)** 에서는 `app` 폴더 안에 `.env` 파일에 이미 있을 수 있어요.
- **Vercel(배포)** 에 올리면, Vercel 쪽에도 **똑같은 값**을 “환경 변수”로 한 번 입력해줘야 해요.  
  그래야 배포된 앱도 Supabase에 연결됩니다.

---

## 1단계: Supabase에서 “주소”와 “키” 찾기

**어디서:** Supabase 웹사이트 → 내 프로젝트

1. [supabase.com](https://supabase.com) 로그인
2. 사용 중인 **프로젝트** 클릭
3. 왼쪽 아래 **톱니바퀴(⚙️)** 클릭 → **Project Settings**
4. 왼쪽 메뉴에서 **API** 클릭
5. 이 화면에서 두 개만 찾으면 됩니다:
   - **Project URL**  
     → `https://xxxxx.supabase.co` 같은 주소. **이게 `VITE_SUPABASE_URL` 값**이에요.
   - **Project API keys** 안에 **anon** / **public** 이라고 된 키  
     → 긴 문자열. **이게 `VITE_SUPABASE_ANON_KEY` 값**이에요.

**정리:**  
- `VITE_SUPABASE_URL` = Project URL (복사)  
- `VITE_SUPABASE_ANON_KEY` = anon public 키 (복사)

이 두 개를 **메모장에 붙여넣어 두었다가**, 나중에 Vercel에서 쓸 거예요.

---

## 2단계 전에: Git 설치하기 (한 번만)

**"git은 내부 또는 외부 명령…"** 이라고 뜨면 → PC에 **Git이 없는 것**이에요. 먼저 설치해야 해요.

1. **다운로드:** [https://git-scm.com/download/win](https://git-scm.com/download/win)  
   → "Click here to download" 클릭해서 설치 파일 받기
2. 설치 파일 실행 → **Next**만 눌러도 됨 (기본값 그대로)
3. 설치 끝나면 **명령 프롬프트를 완전히 닫았다가** 다시 열기  
   (그래야 `git` 명령이 인식됨)
4. 새로 연 명령 프롬프트에서 `git --version` 입력 후 엔터  
   → 버전 번호가 나오면 설치된 거예요.

---

## 2단계: GitHub에 코드 올리기

1. [github.com](https://github.com) 로그인
2. 오른쪽 위 **+** → **New repository**
3. Repository name: 예) `chungju-nursery-app`  
   → **Create repository** (다른 건 건드리지 않아도 됨)
4. **내 PC**에서 아래 명령어 실행하기:
   - **폴더를 탐색기로 따로 열어둘 필요는 없어요.**  
     명령 프롬프트에서 `cd ...` 를 치면 그 폴더로 “이동”한 거예요.
   - **명령 프롬프트**를 연 다음, 아래를 **한 줄 입력하고 엔터 → 다음 줄 입력하고 엔터** 반복하면 됩니다.  
     (본인 GitHub 아이디와 저장소 이름에 맞게 `본인아이디/chungju-nursery-app` 부분만 바꾸세요.)

```bash
cd "c:\Users\PC\Desktop\my first vibe\app"
git init
git add .
```

- **처음 한 번만:** Git이 "누가 커밋했는지" 알 수 있도록 이름·이메일 설정  
  (아래 두 줄에서 `본인이메일@주소`, `본인이름` 을 본인 걸로 바꿔서 실행)

```bash
git config --global user.email "본인이메일@주소"
git config --global user.name "본인이름"
```

- 그다음 커밋부터 진행:

```bash
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/본인아이디/chungju-nursery-app.git
git push -u origin main
```

- `git push` 할 때 **"Connect to GitHub" / "GitHub Sign in"** 창이 뜨면:
  - **Sign in with your browser** (브라우저로 로그인) 버튼을 누르세요.
  - 브라우저가 열리면 GitHub 로그인(또는 이미 로그인돼 있으면 "Authorize"만 누르기) 하면 됩니다.
  - 끝나면 명령 프롬프트 쪽에서 자동으로 푸시가 완료됩니다.
  - 브라우저 로그인이 안 되면 **Token** 탭에서 Personal Access Token을 넣어도 됩니다. (GitHub → Settings → Developer settings → Personal access tokens 에서 토큰 생성)

이렇게 하면 **우리 앱 코드가 GitHub 저장소에 올라간 상태**가 됩니다.

## 3단계: Vercel에서 “배포 사이트에서 설정” 하기

**“배포 사이트에서 아래 두 개를 설정한다”** = **Vercel 웹사이트에서, 우리 프로젝트 설정란에 Supabase 주소와 키를 입력하는 것**이에요.

### 3-1. Vercel에 프로젝트 가져오기

1. [vercel.com](https://vercel.com) 접속 → **Login** → **GitHub**로 로그인
2. **Add New…** → **Project**
3. **저장소 목록이 안 보이면:** "Import Git Repository" 칸 안에 **Install** (GitHub 로고 있는 검은 버튼)을 누르세요. GitHub에서 Vercel 허용 후 다시 Vercel로 돌아와서 **Add New… → Project** 다시 들어가면 저장소 목록이 보입니다.
4. 목록에서 **chungju-nursery-app** (또는 본인 저장소 이름) 선택 → **Import**
5. **Root Directory:** GitHub에 올릴 때 `app` 폴더 안에서 올렸다면 저장소 루트가 이미 앱 폴더라서, **Edit 눌렀을 때 나오는 목록에서 "저장소이름 (root)" 그대로 선택하고 Continue** 하면 됩니다. (별도로 `app` 입력할 칸이 없으면 루트가 맞는 거예요.)
6. **Environment Variables** 라는 펼침 메뉴 있음 → 펼치기

### 3-2. 여기서 “두 개 설정” (이게 26~33번 말한 거예요)

**Environment Variables** 안에 **이름(Name)** 과 **값(Value)** 을 넣는 칸이 있어요.

**첫 번째:**

- **Name:** `VITE_SUPABASE_URL` (그대로 복사해서 입력)
- **Value:** 1단계에서 복사한 **Project URL** (`https://xxxxx.supabase.co`)

**두 번째:**

- **Name:** `VITE_SUPABASE_ANON_KEY` (그대로 복사해서 입력)
- **Value:** 1단계에서 복사한 **anon public 키** (긴 문자열)

각각 입력한 뒤 **Add** 또는 체크 표시 해두고,  
다 적었으면 **Deploy** 버튼 클릭.

- **“무슨 설정을 어디서?”**  
  → **Vercel** 사이트에서, **우리 프로젝트 Import 할 때 나오는 화면**의 **Environment Variables**에서, **Name에 `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY`**, **Value에 Supabase에서 복사한 값** 넣는 게 “배포 사이트에서 두 개 설정”하는 거예요.

배포가 끝나면 **https://프로젝트이름.vercel.app** 같은 주소가 생깁니다. 이 주소가 **우리 앱 주소**예요.

---

## 4단계: Supabase에 “배포된 주소” 알려주기

로그인 등이 배포된 주소에서도 되게 하려면, Supabase에게 “이 주소도 우리 앱이에요”라고 등록해야 해요.

1. **Supabase** 대시보드 → 왼쪽 **Authentication** → **URL Configuration**
2. **Site URL**을 배포 주소로 바꾸기  
   예: `https://chungju-nursery-app.vercel.app`
3. **Redirect URLs**에 **Add URL**로 같은 주소 추가  
   예: `https://chungju-nursery-app.vercel.app/**`  
   → **Save**

이렇게 하면 “배포 사이트에서 설정” + “Supabase에 배포 URL 등록”까지 끝난 거예요.

---

## 5단계: 폰에서 쓰기

- 폰 브라우저 주소창에 **https://우리앱.vercel.app** 입력해서 접속
- **홈 화면에 추가** (아이폰: Safari 공유 → 홈 화면에 추가 / 안드로이드: Chrome 메뉴 → 앱 설치 또는 홈 화면에 추가)  
  → 앱처럼 아이콘 눌러서 쓸 수 있어요.

---

## 배포 실패 시: "Permission denied" / exit 126

빌드 로그에 **Permission denied** 또는 **exited with 126** 이 나오면, 보통 **node_modules**가 GitHub에 올라가서 생긴 문제예요. 아래처럼 정리한 뒤 다시 푸시하세요.

1. `app` 폴더에 **.gitignore** 파일이 있는지 확인 (없으면 만들어 두었음: `node_modules`, `dist`, `.env` 등 무시)
2. 명령 프롬프트에서 `app` 폴더로 이동한 뒤:

```bash
cd "c:\Users\PC\Desktop\my first vibe\app"
git rm -r --cached node_modules
git add .
git commit -m "Add .gitignore, stop tracking node_modules"
git push
```

3. Vercel은 푸시만 되면 자동으로 다시 배포해요. 한 번 더 배포된 뒤 빌드 로그 확인.

---

## 정리 (질문에 대한 답)

- **“배포 사이트에서 아래 두 개를 설정”**  
  → **Vercel**에서 프로젝트 가져올 때 나오는 **Environment Variables** 칸에  
  `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` **이름**으로, Supabase **주소**랑 **anon 키** **값** 넣는 것.

- **VITE_SUPABASE_URL이 뭐고 어디 있는지**  
  → **Supabase** 로그인 → **Project Settings** → **API** → **Project URL**이 그 값이에요.  
  → 이름만 `VITE_SUPABASE_URL`로 해서 Vercel에 적어주면 됩니다.

이 순서대로만 하면, “어디서 뭘 설정하는지” 감이 올 거예요. 막히는 단계가 있으면 그 단계 번호만 알려주면 더 짧게 그 부분만 풀어서 써줄 수 있어요.
